# 简历项目条目｜Career Readiness Agent

**Career Readiness Agent（求职准备智能体）**          **独立开发**                  **2026.07-至今**

**Github 链接：**待发布

**技术栈：**Python、FastAPI、Streamlit、SQLite、LangChain、Chroma、Pydantic、OpenAI 兼容模型 API、pytest

**项目描述：**面向校招求职场景，设计并实现一套本地优先的求职准备智能体。系统将普通聊天与递进式求职流程整合，支持用户上传简历并生成可确认的能力档案，录入和分析多个目标岗位 JD，基于已确认的证据识别能力缺口、给出简历优化建议和可执行的准备任务；对知识库回答、公开联网研究和模型分析分别保留来源或用户授权边界。

**主要职责与技术难点：**

1. **递进式求职工作流与多经历建模：**使用 Streamlit 实现“隐私选择—能力档案—岗位机会—准备任务—求职报告”的可回退流程；围绕项目、实习、竞赛、课程与校园经历建立结构化证据模型，支持同类别经历的新增、编辑、确认和删除，避免单条经历覆盖。
2. **简历解析与模型结构化抽取：**实现 Word/PDF 文本提取与本地规则切分，并在用户同意模型分析后调用结构化输出 API，将简历内容自动填充为待确认经历；针对相邻项目、日期区间和技术栈设置独立切分约束，模型不可用时自动降级为本地解析，保证核心流程可用。
3. **岗位 JD 分析与跨岗位优先级：**通过 FastAPI 将岗位录入、确认、自动需求分析、活跃岗位选择和主岗位设置封装为稳定接口；模型根据用户已确认的能力证据标记岗位要求的匹配、证据不足或能力缺口，并支持最多三个岗位并行推进及下一任务排序。
4. **本地 RAG、会话记忆与可追溯回答：**使用 Chroma 构建文档切片索引，将来源文件、相对路径和切片编号随检索结果返回；以 `thread_id` 和 SQLite 持久化对话记忆，使“展开上一条”“缩短上一条”等上下文指令可生效，并对模型/API 异常返回面向用户的友好提示。
5. **隐私控制、联网研究与工程质量：**将本地处理、模型分析、公开联网研究拆分为可撤销授权；联网研究只发送查询词并按第一方/补充来源分级，个人证据不外发；为 API、简历初始化、岗位分析、规划、RAG 与 Web UI 建立自动化测试，当前全量测试 **71 passed**。

**项目亮点：**在“通用大模型聊天”之外，将用户的真实经历、岗位要求和后续行动闭环到可确认、可回退、可追溯的本地求职工作流，避免模型把未证实内容直接写入简历或建议。
